## 🛡️ Security Principles

> The philosophy behind every system I build.

```
01. LEAST PRIVILEGE       — Every user, service, and API gets only what it needs. Nothing more.
02. SECURE BY DEFAULT     — Systems ship locked down. Open what is needed. Never the reverse.
03. DEFENSE IN DEPTH      — Multiple layers. No single point of failure or trust.
04. AUDIT READINESS       — Logs, traces, alerts from day one. Not added after the breach.
05. INPUT VALIDATION      — Every input is hostile until proven safe.
06. ENCRYPT IN TRANSIT    — TLS everywhere. HTTP is not acceptable in production.
07. ENCRYPT AT REST       — Sensitive data encrypted before it touches the disk.
08. FAIL SECURELY         — Errors should never leak system internals or sensitive data.
09. ZERO TRUST MINDSET    — Verify everything. Trust nothing by default, inside or outside.
10. PATCH AGGRESSIVELY    — Unpatched systems are open invitations.
```
